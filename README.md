# Запуск

Для запуска используется docker-compose.yml файл.

Для запуска необходимо:
```
docker compose up -d --build
```

# Запуск тестов

```
mvn test
mvn verify
```

После этого будет запущено 2 контейнера: с postresql и с приложением.

# Доступно для просмотра и тестов

Документация в swagger: http://localhost:8080/swagger-ui/index.html

Коллекция Postman: PostmanRequests/Task Time Tracker API.postman_collection.json

# Что реализовано

Реализованы REST-эндпоинты согласно техническому заданию: 
 - создание задачи, 
 - получение задачи по идентификатору, 
 - изменение статуса задачи (NEW, IN_PROGRESS, DONE), 
 - создание записи о затраченном времени сотрудника на задачу, 
 - получение списка записей времени сотрудника за заданный временной период. 

Используется база данных – PostgreSQL, запускаемая в Docker‑контейнере. 

Выполнены дополнительные критерии: 
- декларативная документация API через SpringDoc OpenAPI (Swagger UI доступен по адресу http://localhost:8080/swagger-ui/index.html), 
- входные DTO валидируются с помощью Bean Validation, 
- исключения обрабатываются глобальным обработчиком @RestControllerAdvice, 
- написаны интеграционные тесты DAO‑слоя с Testcontainers, 
- написаны unit‑тесты сервисов с использованием JUnit 5 и Mockito. 

Проект включает готовую коллекцию Postman для тестирования эндпоинтов 
инструкции по сборке и запуску через Docker Compose. 



# Использование ИИ
ИИ использовался для:
1) генерации набора тестовых запросов, демонстрирующих работу всех реализованных REST-эндпойнтов
Промт: Приложен код контроллеров, напиши запросы, которыми можно проверить работу всех реализованных REST-эндпойнтов
2) генерация аннотаций для документации
3) Промт: Приложен код контроллеров, напиши аннотации для генерации документации по доступным эндпойнтам для Swagger
